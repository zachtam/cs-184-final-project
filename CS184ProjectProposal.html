Resonance Simulation Via the Chladni Plate Experiment

Overview:

We want to replicate resonance demonstrations like the Chladni Plate Experiment (see https://www.youtube.com/watch?v=wvJAgrUBF4w) using the simulation techniques we learned in class. If we have time, we hope to extend to visualizing with water rendering instead of sand particles.

Problem Description:

In this project, we want to be able to demonstrate the convergent behavior of particles in a resonant environment (e.g., on a vibrating plate). This is an interesting demonstration that could be used for a cool website or teaching tool, and also serves to stress-test our models of particle and plate behavior. The challenge in the project comes from simulating the actual non-rigid behavior of a plate of sheet metal and combining this with accurate simulation of thousands of particles of sand or water. If either of these are missing, we will not observe the local convergence to resonance patterns like in real life. Furthermore, if we want to be able to watch the progression in real time, we will likely need to CUDA-accelerate our process.

To solve this problem, we plan to (1) model the plate as very stiff cloth using the same strategy as used in homework 4, and (2) model the particles with inspiration from existing particle physics simulations. Pinning the plate in the center and applying an oscillating force, we should be able to simulate the true behavior in the Chladni Plate Experiment.

Goals and Deliverables:

Through this project, we are trying to create the behavior of sand particles on a vibrating plate. Specifically, we aim to create the vibrating plate using the stiff cloth simulation used in homework 4, and model the sand using particle physics simulations. Following this, applying an oscillating force on the vibrating plate will allow us to perform the simulation.

We aim to test this with a range of tests: the simplest being making sure the simulation behaves in the same way as the inspiration video of the experiment being run in real life (https://www.youtube.com/watch?v=wvJAgrUBF4w). We think this is a feasible project to implement because all techniques required we have learnt in class or we have already explored online and understand how to implement.

We plan to create a demo showing a set of patterns (such as in the inspiration video) we prepare using the analytical methods in the resources section (we’ve found resources that detail how to predict the patterns with interesting math). Beforehand, we will specify the simulations we aim to demonstrate with accompanying math and an underlying explanation as to how it works, and then demonstrate the patterns with our rendering. We aim to measure the quality of our system by testing to see whether our patterns match the pre-specified analytical solution. To do this, we will check the particle density across the image and make sure it aligns with the analytically expected distribution given the same oscillation frequency.

Through our analysis, we want to show that we can execute high-enough fidelity particle and cloth simulation using the techniques learned in class to accurately demonstrate the complex effects of resonance.

Our “baseline plan” would be, given an input frequency, time horizon, and number of sand grains, to produce a “final” sand positions image. We would simulate the motions of the sand particles offline, and show only their final positions at the end time horizon as our result. In the simplest case, we would render simply top-down 2D with no complex lighting or raytracing.

Our “aspirational plan” would include the following extensions:

  • rendering in the homework 4 visualizer (3D, can adjust viewing angle)
  • visualizing the motion of the sand in real time as the vibration changes, or as we click-to-pour more sand onto the sheet metal (i.e., interactive demo!)
  • rendering different materials for the substrate
  • rendering different granular media (sand, water, fire https://www.youtube.com/watch?v=2awbKQ2DLRE)
  • CUDA-accelerating our project as necessary to make it function in real-time (particularly as the rendering gets more complicated)
